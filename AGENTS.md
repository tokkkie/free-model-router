## 🚨 STOPPERS（即時停止・禁止事項）

| 禁止事項 | 違反時の対応 |
|---------|------------|
| **無断の追加修正** | 「ついでに」の修正禁止。必ず提案→承認 |
| **計画なき独走** | 作業開始前に「実施手順」を提示し承認を得よ |
| **エラー無視** | コマンド失敗（Exit != 0）は即停止。無視して続行を禁ずる |
| **仕様の推測実装** | 不明点を推測で補完することを禁止。必ず確認せよ |
| **環境/Git制約違反** | **`.githooks/` のエラーログに従い、自律的に修正・再試行せよ** |
| **Public化禁止** | プライベートリポジトリをPublicに変更することを厳禁とする |

## 🛠️ WORKING PROTOCOL（作業指針）

### 1. 作業開始前（必須）
以下を簡潔に提示し承認を得てから着手せよ：
- **目的 / 変更対象**: 何を、どのファイルで達成するか
- **手順 / 影響範囲**: 最小差分での手順と、他に及ぼす影響

### 2. 不明時・不確実時の動作
- **不明点**: 作業を中断し確認を求める。「不明」と明示せよ
- **提案モード**: 不確実性が高い場合は実装せず、メリット/デメリットを伴う選択肢を提示せよ
- **例外**: typo修正、コメント修正、未使用import削除のみ事前承認なしで実施可能（事後報告必須）

### 3. セキュリティ・リカバリ
- **権限**: 最小権限で動作せよ。不要な権限昇格を禁ずる
- **破壊的操作**: DB操作や一括削除前はロールバック手段（バックアップ等）を確認せよ
- **インシデント時**: 即座に情報を秘匿化し、影響範囲を特定の上、ユーザーに即時報告せよ

## 💻 TECHNICAL SPEC（技術仕様）

- **Language**: 対話・コメント・コミット・PRは**日本語を基本**とする。
- **Git Hooks**: 文字コード、改行コード、命名規則、ブランチ操作、コミット形式、シークレット混入等は **Hook で物理的に強制される。**
- **最小差分主義**: 10行以上、または複数ファイルの変更は必ず `diff` または `patch` 形式で提示せよ。
- **依存追加**: 理由と代替案を提示し承認を得ること。

## ✅ DONE（完了定義）
報告前に以下を自律確認せよ：
- [ ] ビルド・テスト成功
- [ ] 差分確認（`git diff`）済み
- [ ] 意図しない変更・機密情報の混入なし
- [ ] 構成変更時は `README.md` の構造図を更新済み
- [ ] 冪等性の確保（再実行しても安全か）
